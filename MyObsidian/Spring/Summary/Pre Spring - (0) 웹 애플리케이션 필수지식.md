## 웹페이지가 나오기까지...
- **클라이언트**는 어떠한 서비스를 요청하는 쪽, **서버**는 어떠한 서비스를 제공하는 관계.
= 손님과 알바생이 되겠다
- **브라우저** : 인터넷을 이용할때 이용하는 소프트웨어 = 프랜차이즈가 되겠다
- **HTTP** : 클라이언트와 서버가 요청하고 서비스를 하는 것(대상) = 음료주문이 되겠다

<br>

### HTTP 통신과정
= 음료를 만드는 과정
<br>
1. 클라이언트(손님)가 브라우저에 URL 입력 = 원하는 프랜차이즈가서 키오스크로 어떤 음료를 주문할건지 입력
2. URL이 인터넷으로 웹서버에 HTTP 요청이 전송 = 음료 주문한게 알바생에게 전달
3. 웹 서버는 HTTP요청을 받아 해당 HTTP를 클라이언트로 전송 = 알바생이 음료 뚝딱 만듦
4. 브라우저는 받은 응답을 표시하고 클라이언트가 확인 = 진동벨이 울려서 손님이 시킨 음료가 맞는지 확인& 픽업

## GET 메서드 & POST 메서드
:  HTTP의 요청의 한 유형 = 키오스크로 주문할건지 그냥 주문할건지 오더로 주문할건지... 요청유형!

- GET메서드 : 브라우저를 통해서 서버에게 값을 전달 할 때, URL 뒤에 값을 더하여 보내는 방식 = `쿼리 스트링 또는 쿼리 문자열`
	- 쿼리 스트링 특징
		1. URL 끝에 ? 가 붙어서 시작을 알림 ex) domain.com/?
		2. 형식은 "이름 = 값" ex) domain.com/?name=value
		3. 여러 값을 전달하려면  &이용 ex) domain.com/?name=value&tel=010
		4. 하지만 많은 양의 값을 보내기에 적합하진 않음 = 알바생 2명인데 음료 계속 40개 쳐내는거 같은...
	- POST 메서드 : 요청 본문이라는 URL에 보이지 ㅇ낳는 장소에 값을 넣어서 보내는 방법
